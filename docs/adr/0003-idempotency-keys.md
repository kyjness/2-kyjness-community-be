# ADR 0003 — POST 멱등성: Idempotency-Key + 결과 캐시 + 진행중 락

- **상태**: 채택됨 (Accepted)
- **관련 코드**: `app/api/dependencies/client.py`

## 맥락 (Context)

네트워크 타임아웃 시 클라이언트는 같은 POST를 재시도한다. 방어가 없으면:

- 게시글이 **중복 생성**되거나 이미지가 **중복 업로드**된다.
- 동일 요청이 동시에 두 번 들어와 경합한다.

결제 수준은 아니지만, **상태를 생성하는 POST**에서는 재시도 안전성이 필요하다.

## 결정 (Decision)

`X-Idempotency-Key` 헤더 기반 멱등성을 **상태 생성 POST 3곳**(게시글 생성·이미지 업로드·가입 이미지 업로드)에 적용한다.

1. **지문화**: `(스코프, 키)`를 sha256으로 fingerprint — 같은 키라도 사용자/용도가 다르면 분리.
2. **결과 캐시**: 최초 성공 응답을 Redis에 저장(TTL). 같은 키 재요청은 캐시된 응답을 그대로 반환.
3. **진행중 락**: 처리 중인 키에 대한 동시 요청은 `409`로 차단(중복 실행 방지).
4. **fail-open**: Redis 오류 시 멱등 보호를 건너뛰고 정상 처리(가용성 우선).

## 트레이드오프 (Consequences)

**얻은 것**: 재시도·더블클릭 안전, 중복 리소스 생성 차단. 적용 범위를 POST 3곳으로 한정해 오버헤드 최소.
**치른 비용**:
- **Opt-in** — 클라이언트가 키를 보내야 효력. (키 없으면 일반 처리)
- **TTL 윈도 한정** — 영구 멱등이 아니라 캐시 TTL 동안만 보장(요구엔 충분).
- **Redis 의존**(fail-open으로 완화).

## 고려한 대안 (Alternatives)

| 대안 | 기각 사유 |
|------|-----------|
| DB unique constraint만 | 자연 키 없는 생성(게시글 등)엔 요청-단위 멱등을 보장 못 함 |
| 무방어 | 재시도 시 중복 생성 그대로 노출 |
| 전 엔드포인트 멱등 적용 | 조회·idempotent 메서드엔 불필요 — 상태 생성 POST에만 |

## 일부러 하지 않은 것 (Non-goals)

- 영구 멱등성 / 정확히-한-번 전역 보장 — 재시도 윈도 방어로 충분.
- GET·PUT 등 본래 idempotent하거나 비생성 경로엔 미적용.
